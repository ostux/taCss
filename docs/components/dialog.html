About
Dialog

The dialog component allows for verification of user actions, simple data input, and alerts to provide extra information to users.

Actions order

The order is automatically reversed for actions. So, the first action you create will appear last on the action bar.

Responsible dialog and allert width

On phone screen the dialog width: 90% fixed, or 100%-gutter*2. Up to 3/2 part of tablet screen the dialog width: 80% fixed or 100%-20%-gutter*2. Otherwise the dialog wide what you request within the dialog class. Don't forget to use no-gutter class if you dont want gutter around the dialog.

dialog-content scrollable. The dialog max height: 90% of the screen. After the content srollable. So feel free to use longer lists inside. Dialog can use even for alert as well...

Predefined styles

For styling, (booth, with javascript or without) you can use one of the five predefinied classes ( .primary, .success, .info, .warning, .danger)
To use the predefinied classes require to set $enable-variants: true !default in base/_variations.sass!
